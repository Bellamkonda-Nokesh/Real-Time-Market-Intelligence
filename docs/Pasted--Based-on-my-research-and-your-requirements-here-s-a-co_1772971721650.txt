## 3. Technical Architecture

### 3.1 Stack Selection (Optimized for Simplicity + Performance)

| Layer | Technology | Justification |
|-------|-----------|---------------|
| **Backend** | **FastAPI** (replaces Flask) | 3x faster, automatic API docs, async support  |
| **Database** | **SQLite** (production-hardened) | Single file, zero config, sufficient for student scale  |
| **Frontend** | **Vanilla JS + SPA architecture** | No framework overhead, full control, fast  |
| **Styling** | **Tailwind CSS** | Rapid development, consistent design system |
| **Charts** | **Chart.js** | Lightweight, beautiful, easy to customize  |
| **Animations** | **GSAP + CSS transitions** | 60fps performance, professional feel |
| **Icons** | **Phosphor Icons** | Modern, consistent iconography |
| **Fonts** | **Inter + JetBrains Mono** | Highly readable, professional aesthetic |

### 3.2 Why FastAPI over Flask?

- **Performance**: 3x faster request handling 
- **Type Safety**: Pydantic validation prevents runtime errors
- **Auto Documentation**: `/docs` endpoint generates interactive API docs
- **Async/Await**: Handle multiple NewsAPI calls concurrently
- **Modern Python**: Built for Python 3.7+ with latest features

### 3.3 SQLite Production Configuration

```python
# Optimized SQLite for concurrent reads
DATABASE_URL = "sqlite:///./intelligence.db?mode=ro"  # Read-only mode for queries
# Write operations use separate connection with WAL mode
```

**Optimizations** :
- WAL (Write-Ahead Logging) mode for concurrent read/write
- Connection pooling with `sqlite3` thread safety
- Indexed columns: `timestamp`, `sentiment_score`, `company_name`
- JSON column support for flexible metadata storage

### 7.2 Optimization Strategies

**Backend**:
- Async database queries with `aiosqlite`
- Connection pooling (max 10 connections)
- Response caching with `fastapi-cache` (Redis-like in-memory)
- Compression middleware (gzip/brotli)

**Frontend**:
- Lazy loading for below-fold content
- Image optimization (WebP format, lazy loading)
- Code splitting by route
- Service Worker for offline caching
- Debounced search inputs (300ms delay)

**Database**:
- Indexed columns for frequent queries
- Query result caching (5-minute TTL)
- Batch inserts for news ingestion
- WAL mode for concurrent access

## 8. Data Architecture

### 8.1 Database Schema (SQLite)

```sql
-- Users table
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT UNIQUE NOT NULL,
    hashed_password TEXT NOT NULL,
    full_name TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT 1
);

-- News articles
CREATE TABLE articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    content TEXT,
    source TEXT,
    url TEXT UNIQUE,
    published_at TIMESTAMP,
    fetched_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    raw_sentiment REAL,
    processed_sentiment REAL,
    sentiment_label TEXT CHECK(sentiment_label IN ('positive', 'negative', 'neutral')),
    keywords TEXT -- JSON array
);

-- User watchlists
CREATE TABLE watchlists (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER REFERENCES users(id),
    company_name TEXT NOT NULL,
    keywords TEXT,
    alert_threshold REAL DEFAULT 0.2,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Sentiment history (for charts)
CREATE TABLE sentiment_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date DATE NOT NULL,
    avg_sentiment REAL,
    article_count INTEGER,
    positive_count INTEGER,
    negative_count INTEGER,
    neutral_count INTEGER
);

-- Indexes for performance
CREATE INDEX idx_articles_date ON articles(published_at);
CREATE INDEX idx_articles_sentiment ON articles(sentiment_label);
CREATE INDEX idx_sentiment_history_date ON sentiment_history(date);
```

## 9. Security Requirements

### 9.1 Authentication & Authorization

- **JWT Tokens**: Access token (15 min expiry) + Refresh token (7 days)
- **Password Hashing**: bcrypt with salt rounds=12
- **HTTPS Only**: Secure cookie flags, HSTS headers
- **Rate Limiting**: 100 requests/minute per IP, 1000 per user
- **CORS**: Whitelist specific origins only

### 9.2 Data Protection

- **Input Validation**: Pydantic models for all endpoints
- **SQL Injection Prevention**: Parameterized queries only
- **XSS Protection**: Content Security Policy headers, output encoding
- **API Key Security**: NewsAPI/Gemini keys in environment variables only

### 9.3 Error Handling

- **Graceful Degradation**: If NewsAPI fails, show cached data with warning
- **User-Friendly Errors**: No stack traces exposed to frontend
- **Logging**: Structured logging (JSON format) for debugging
- **Health Checks**: `/health` endpoint for monitoring
